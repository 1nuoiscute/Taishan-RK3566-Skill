# 电赛视觉任务类型

根据题目实际约束加载对应章节。一个题目可以同时属于多个类型。

## 静态目标测量

代表案例：2025 C 题“基于单目视觉的目标物测量装置”。

先确定目标平面、参考尺寸、相机姿态、畸变、测量量、尺度模型和误差指标。优先建立 ROI、边缘/轮廓/标志点和像素到物理量的映射，再讨论复杂模型。

验收至少区分单帧测量误差、连续测量稳定性、不同位置/姿态误差和端到端输出延迟。没有标定板、参考尺寸或题目允许的场景条件时，不声称可以直接测出真实距离。

## 运动目标追踪

代表案例：2023 年“运动目标控制与自动追踪系统”。

先定义检测、关联、跟踪、丢失、重捕获和退出状态。记录采集帧率、处理帧率、端到端延迟、目标速度、遮挡时长和误差。只有在运动模型、延迟预算和题目指标支持时，才引入预测；不要把单帧检测结果当作稳定跟踪。

降级路线通常包括降低搜索区域、缩小输出频率、保留最后可信状态、重新全局搜索和安全停止。所有控制输出必须带时间戳、有效性和失效动作。

## 视觉控制与瞄准

代表案例：2025 E 题“简易自行瞄准装置”的视觉子任务。

把链路拆成：靶面/目标识别 -> 靶心或目标点定位 -> 相机坐标 -> 云台/执行机构坐标 -> 控制输出 -> 光斑/结果反馈。明确相机内参、安装姿态、云台零位、正负方向、单位、限幅和急停。

若巡迹、电机或安全控制由 MSPM0 等下位机完成，不把这些职责无理由迁移到 RK3566。先验证固定目标定位和低速单轴动作，再做二维闭环和移动期间连续输出。

激光、云台和电机测试必须设置限幅、空载/低速模式和可观察的停止条件。题目给出的距离误差、时间限制和同步误差必须进入验收表，不能只报告识别准确率。

## 空间场景与多目标盘点

代表案例：2024 年“立体货架盘点无人机系统”。

先拆分视角变化、目标类别、目标数量、空间层级、遮挡、重复目标和盘点完成条件。区分“看到目标”“识别目标”“估计位置”“完成盘点”四个状态。若没有深度传感器，不把单目图像直接当作可靠三维距离；可使用已知尺寸、平面约束、多视角或题目允许的运动信息建立可验证的近似。

验收应记录漏检、重复计数、误计数、位姿/距离误差、视角变化、飞行/平台运动期间的延迟和安全降级。

## 传统视觉识别

适用于颜色、线、几何轮廓、标志、二维码或规则图案明确的题目。优先选择可解释、可标定、可降级的 ROI、阈值、形态学、边缘、轮廓、拟合和时序滤波组合。

当光照、背景、外观或类别语义使传统路线无法稳定满足指标时，再评估 OpenCV DNN、RKNN 或其他模型运行时。模型方案必须包含数据、版本、格式、授权、量化、输出解码、目标板延迟和无模型降级说明。

## 结构化标记、格点与路径

适用于编码标记、规则格点、棋盘、赛道边界或路径中心线。先确定方向、拓扑、缺失元素、交叉/岔路和无效输出，再选择解码、角点、轮廓、线段或区域拟合。不要把单帧找到局部线段直接等同于已经获得稳定路径或正确格点顺序。

最小验证使用固定图像或用户提供的录制序列，输出原始候选、排序/方向、质量量和无效原因。涉及运动控制时，先离线验证偏差符号和丢失状态，再独立验证执行接口；路线不稳定时可缩小有效区域、降低输出粒度或请求重新观测。

## 混合题型

题目同时包含识别、测量、跟踪或控制时，为每个子任务分别定义输入、输出和通过条件，再确定依赖顺序。例如“先确认目标有效，再计算坐标”是依赖关系，不代表两步必须使用同一种算法。只在前一阶段证据满足门槛后进入下一阶段，并为每个阶段保留独立降级输出。
