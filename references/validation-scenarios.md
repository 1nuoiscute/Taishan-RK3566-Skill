# Skill 回归场景

修改 Skill、模板或参考资料后，至少执行一个相关场景，记录输出是否满足断言。

## 通用题目输入

**输入**：只给一份视觉赛题和一句“帮我做”。

**必须出现**：题目约束卡、已确认/待确认区分、视觉类型、最小验证和不擅自固定的协议/摄像头/模型假设。

## 低风险 API 或硬件测试

**输入**：询问如何检查摄像头、UART 或 GPIO。

**必须出现**：探测命令或最小测试、预期现象、失败排查；不得直接生成完整闭环工程。

## 静态测量

**输入**：2025 C 类单目测量题。

**必须出现**：标定/参考尺寸/误差分析；不得只报告识别准确率或假定单目直接得到深度。

## 运动跟踪

**输入**：2023 运动目标题。

**必须出现**：检测、关联、丢失、重捕获、延迟和状态；不得把单帧检测当作稳定跟踪。

## 视觉瞄准

**输入**：2025 E 视觉子任务。

**必须出现**：目标点、坐标转换、云台接口、时间/距离误差、安全和下位机职责边界。

## 模型不足

**输入**：要求 YOLO，但没有模型、数据集或目标板运行时。

**必须出现**：缺口清单、数据/部署计划、传统视觉或最小采集降级；不得声称已经可训练或可部署。

## 已有工程

**输入**：提供现有代码和串口协议，要求增加视觉模块。

**必须出现**：先读现状清单、保留已确认接口、局部修改和回归测试；不得平行重写。

## 性能不足

**输入**：用户报告帧率低或闭环卡顿。

**必须出现**：采集/处理/发布/端到端口径、证据、瓶颈假设、优化代价和回归测试。
