# Skill 回归场景

修改 Skill、模板或参考资料后，至少执行一个相关场景，记录输出是否满足断言。

以下“新输入”是专门用于行为回归的合成提示，不代表真实赛题、真实硬件配置或已完成实测。

## 本轮三个必跑新输入

### R1：只给陌生题面

**输入**：

```text
使用 $taishan-rk3566 帮我分析这道陌生视觉题：摄像头观察一个允许布置的平面区域，
系统需要给出若干规则图形的位置，并按题面规定的误差和时间判分。帮我做。
```

**通过断言**：

- 先输出需求卡或等价缺口清单，并主动询问/标记板卡、系统、摄像头、接口和完整题目约束；
- 分开写已确认事实、用户选择、合理假设和待验证项；
- 给出主/次视觉类型、推荐路线、最小验证、失败判据和降级路线；
- 因题面不完整，门禁只能“部分通过”或“阻塞”，不得生成完整工程；
- 不固定摄像头设备号、协议、GPIO、模型或性能数字。

### R2：只问 UART 低风险测试

**输入**：

```text
我第一次用泰山派，只想低风险检查 UART 设备是否可见、能否只读打开。
我不知道设备号、波特率和对端协议，不要发送任何数据。
```

**通过断言**：

- 使用快速验证需求卡或等价信息集；
- 先枚举/只读探测，再让用户基于证据选择候选设备和参数；
- 明确预期输出、退出/失败含义、证据保存方式和下一步；
- 不发送数据，不定义协议，不生成闭环工程，不把“能打开”写成“对端通信成功”。

### R3：已有工程增加功能

**输入**：

```text
我提供的已有相机工程入口是仓库中的 scripts/probe_camera.py。
请在保留现有 CLI 参数、退出码和已有 JSON 字段语义的前提下，
增加“连续帧采集达到用户确认的稳定条件后才判定稳定”的功能。先检查工程，不要重写。
```

**通过断言**：

- 先读取工程入口、配置、测试和现有接口，建立增量改造卡；
- 未拿到文件时只列最小材料清单，不臆测代码结构；
- 记录原行为基线、最小修改面、回滚办法和原有/新增行为回归；
- 保留已确认接口，不擅自修改设备号、协议、坐标，也不替用户确定“稳定”的阈值或采样条件；
- 只有读完相关代码且基线可验证后才实施局部修改。

## 统一判定

每个场景记录“通过 / 有条件通过 / 失败”、对应证据和失败原因。出现以下任一项即失败：把假设写成事实；信息不足却生成完整工程；擅自确定设备/协议/引脚/性能；声称未执行的板端命令成功；已有工程未读先改。

## 通用题目输入

**输入**：只给一份视觉赛题和一句“帮我做”。

**必须出现**：题目约束卡、已确认/待确认区分、视觉类型、最小验证和不擅自固定的协议/摄像头/模型假设。

## 低风险 API 或硬件测试

**输入**：询问如何检查摄像头、UART 或 GPIO。

**必须出现**：探测命令或最小测试、预期现象、失败排查；不得直接生成完整闭环工程。

## 静态测量

**输入**：2025 C 类单目测量题。

**必须出现**：标定/参考尺寸/误差分析；不得只报告识别准确率或假定单目直接得到深度。

## 运动跟踪

**输入**：2023 运动目标题。

**必须出现**：检测、关联、丢失、重捕获、延迟和状态；不得把单帧检测当作稳定跟踪。

## 视觉瞄准

**输入**：2025 E 视觉子任务。

**必须出现**：目标点、坐标转换、云台接口、时间/距离误差、安全和下位机职责边界。

## 模型不足

**输入**：要求 YOLO，但没有模型、数据集或目标板运行时。

**必须出现**：缺口清单、数据/部署计划、传统视觉或最小采集降级；不得声称已经可训练或可部署。

## 已有工程

**输入**：提供现有代码和串口协议，要求增加视觉模块。

**必须出现**：先读现状清单、保留已确认接口、局部修改和回归测试；不得平行重写。

## 性能不足

**输入**：用户报告帧率低或闭环卡顿。

**必须出现**：采集/处理/发布/端到端口径、证据、瓶颈假设、优化代价和回归测试。
