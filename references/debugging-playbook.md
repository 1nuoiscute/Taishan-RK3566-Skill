# 调试顺序

## 0. Remote-SSH 与运行身份

确认 VS Code 当前主机、用户、项目路径、权限、解释器、编译器和启动命令。确保日志来自泰山派，而不是本地终端或另一台服务器。

## 1. 相机和光照

确认设备节点、像素格式、曝光、白平衡、分辨率、实际帧率和连续采集。保存原始帧或最小诊断图。

## 2. ROI、阈值与几何

先检查 ROI 和中间图，再检查阈值、形态学、边缘、轮廓、拟合和标定。用固定输入复现误检，不边看闭环输出边调算法。

## 3. 模型与数据

确认模型文件、格式、运行时、输入预处理、输出解码、量化和版本。记录加载时间、推理时间、有效输出和失败日志。没有授权模型或数据集时，使用传统视觉验证或明确阻塞。

## 4. 串口与协议

先做单独收发和解析测试，再接视觉；记录原始字节、时间戳、帧边界、校验、超时和下位机反馈。不要在排查阶段同时改变协议和视觉坐标。

## 5. 闭环、状态机与安全

从固定目标、单轴、低速、限幅和空载开始。检查有效性、时间戳、丢失、超时、急停和恢复状态。

## 6. 性能与崩溃

区分采集、处理、发布和执行频率；记录 CPU、内存、NPU、串口积压、卡顿、重启和长时间运行。先定位瓶颈，再给带代价和回归测试的优化方案。
