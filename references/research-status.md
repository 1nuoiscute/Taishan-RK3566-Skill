# 研究状态

## 已完成资料收集

- 已参考 MaixCAM-skill 的组织方式和工程化工作流。
- 已确认泰山派 RK3566 官方 Wiki 总入口及系统、SDK、工具和文档分类。
- 已记录用户提供的泰山派原理图和官方 40 针 IO 分配表。
- 已记录用户实际工作流：VS Code + Remote-SSH 直连泰山派，在 VS Code 中完成编辑、编译、运行和日志查看。
- 已收集 2025 E、2025 C、2023 运动追踪、2024 立体货架盘点四类赛题案例。
- 已区分官方硬件资料、官方 Wiki 收录的“共建”案例、竞赛题目页面和用户本地资料。
- 已获得一套用户提供的泰山派板端实测基线：Tspi V10、Ubuntu 20.04.6、4.19.232、aarch64、OpenCV 4.13.0，摄像头 `640x480/YUYV` 短时实测约 26.66 FPS。
- 已确认该基线只代表当前用户镜像、接线和摄像头选择；不将 `/dev/video9`、`/dev/ttyS7`、分辨率或性能数字写成通用默认值。

## 暂不做的事情

- 不把官方镜像下载页当作用户实际工作流。
- 不把 ADB、scrcpy、VirtualBox 或 Docker 设为必需环境。
- 不假定固定摄像头、固定 Linux 发行版、固定 V4L2 设备节点或固定串口协议。
- 不把某一份赛题直接写成固定代码模板。
- 不把未经当前镜像和内核验证的 GPIO 数字、摄像头 API 或性能数据写成事实。
