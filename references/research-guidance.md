# 在线研究与来源分级

## 触发条件

题目规则、硬件连接、Linux/内核/API 版本、模型运行时、摄像头能力或候选路线存在明显不确定性时，先查本地资料和用户材料，再查官方来源。用户明确禁止联网时跳过在线研究；浏览不可用时给出关键词和待确认假设，不伪造检索结论。

## 来源分级

1. **原始题面和评分标准**：确认任务、时间、误差、安全和评分事实。
2. **泰山派官方原理图、IO 表、官方 Wiki 和 SDK**：确认硬件、系统、设备树、工具和版本事实。
3. **官方 Wiki 的共建案例**：补充实践路线、命令和失败模式；不能单独证明统一镜像能力或性能。
4. **公开项目、教程和演示**：只作为候选思路，必须在当前板端重新验证。

不得用搜索结果覆盖用户已确认的接线、协议、安全约束或竞赛规则。不得直接复制社区代码，不下载或声称拥有未授权的模型、数据集或资源。

## 研究卡

在方案比较前用简短研究卡记录：

```text
问题：
来源：
来源等级：原始题面/官方/共建/公开经验
来源支持的事实：
仅可参考的经验：
对方案的影响：
未证实项：
下一步验证：
```

## 检索关键词

按板卡、系统和问题组合搜索，例如：`泰山派 RK3566 V4L2`、`RK3566 device tree GPIO`、`RKNN runtime version`、`泰山派 VS Code Remote-SSH`。检索结果只进入候选和验证计划，不直接进入最终结论。
