# 已有工程与联调接入

用户提供源码、设备树、接线、协议、模型或运行日志时，先读取 `templates/existing-project-change.md`，建立现状清单：入口、启动命令、目录、系统、设备、端口、波特率、协议、坐标、线程/循环、已验证项和当前故障。

保留用户已确认的接口和安全联锁，只追问会改变本次方案的缺项。先做局部修复和最小验证，不平行重写整个工程。

接入顺序：

1. 原工程能否在当前 VS Code Remote-SSH 环境启动；
2. 输入、视觉、通信和执行是否能各自观测；
3. 本次改动影响哪个接口和验收指标；
4. 只改变一个主要变量完成回归；
5. 再合并到闭环。

若原工程中已有协议、引脚、坐标或模型，不因“常见做法”自动替换。若内容不足以证明板端事实，标记为待确认并输出探测步骤。

只有在读取相关源码和配置、记录原工程基线并定位最小修改入口后，才开始改动。若用户只描述“有一个工程”但没有提供可读文件或关键片段，先返回增量改造卡和最小材料清单，不生成平行工程。
