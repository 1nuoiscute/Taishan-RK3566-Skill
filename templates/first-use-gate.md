# 首次使用门禁

首次处理一名用户、一个新题目或一个环境不明的已有工程时填写。门禁用于决定下一步，不要求用户一次提供所有答案。

| 板卡型号/版本与 RK3566 职责 |  |  |  |  |
| 系统、内核、架构与开发入口 |  |  |  |  |
| 摄像头、镜头、接口与采集能力 |  |  |  |  |
| UART/GPIO/网络/下位机/执行机构 |  |  |  |  |
| 题目目标、评分、时限、误差与安全 |  |  |  |  |
| 已有代码、协议、标定和实测证据 |  |  |  |  |

“合理假设”必须写明用途、影响和验证方法；不得用它替代设备号、引脚、协议、坐标系、安全动作或性能结论。

## 按需探测

- 基础必查：板卡身份、系统/内核/架构、Codex 是否能访问板端、题目或本次测试目标。
- 涉及图像输入时：确认摄像头来源、接口、候选设备和最小采集证据。
- 涉及 UART 或 GPIO 时：只做可见性/只读探测；写入、发送、申请 line 前必须确认对象和风险。
- 涉及模型时：只在方案确实需要时确认现有模型、授权和板端运行时；缺失时保持为阻塞或采用非模型降级路线。
- 涉及已有工程时：先记录入口、启动方式、基线行为和必须保留的接口，再讨论改动。

## 门禁结论

选择且只选择一种：

- **通过**：足以进行当前阶段；列出支撑证据和仍不影响当前阶段的待验证项。
- **部分通过**：只能执行列出的最小验证；验证返回前不生成完整工程或闭环。
- **阻塞**：缺少会改变方案或安全边界的信息；只输出需求缺口、低风险探测和用户需要确认的选择。
