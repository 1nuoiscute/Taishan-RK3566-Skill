# 泰山派 RK3566 视觉工程结构

按真实职责生成文件，不为“看起来完整”创建空模块。

## 最小工程

```text
project/
  main.py 或 main.cpp       # 唯一入口
  config.*                  # 只有参数确实需要集中管理时创建
```

## 有独立风险时再拆分

```text
project/
  main.py 或 main.cpp
  capture.*                  # 摄像头采集和资源释放
  vision.*                   # 检测、测量、跟踪或模型推理
  geometry.*                 # 标定、坐标和误差转换
  communication.*            # 仅在通信真实存在且协议已确认时创建
  actuator.*                 # 仅在执行机构真实存在时创建
  config.*
  tests/                     # 仅在有可重复测试输入时创建
  models/                    # 仅在用户提供授权模型时创建
```

## 交付要求

- 说明每个文件存在的理由和依赖来源；
- 使用项目内相对路径；
- 提供最小启动命令和可观察的首启动输出；
- 先让输入、算法、接口分别可验证，再启用闭环；
- 缺少模型、摄像头、协议或设备节点时输出降级模式和缺失清单，不声称工程已完成；
- 不把电脑绝对路径、账号、密钥、令牌或未经授权资源写入工程。
